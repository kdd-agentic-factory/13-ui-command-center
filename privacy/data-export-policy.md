# Data Export Policy

## Purpose

This policy governs the export of data outside the ecosystem.

## Export Categories

| Classification | Export Allowed | Requires Approval |
|---|---|---|
| public | yes | no |
| internal | yes, within team | no |
| restricted | yes, with justification | yes |
| sensitive | only for legal/compliance | yes, data owner |

## Export Process

1. Requester identifies dataset and purpose.
2. Data classification is verified.
3. Minimization review is performed.
4. Approval is requested if required.
5. Export is executed with audit event.
6. Export record is stored.

## Paper Export

Data exported for paper publication must be anonymized where possible and documented in the paper evidence package.
