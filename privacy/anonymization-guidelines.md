# Anonymization Guidelines

## When to Anonymize

- Before exporting data for paper publication.
- When sharing data outside the project team.
- When data contains personal information.

## Techniques

### Pseudonymization

Replace identifying values with a pseudonym. Pseudonymization is reversible and does not constitute full anonymization under GDPR.

### Aggregation

Replace individual data points with aggregated statistics.

### Data Masking

Mask specific fields (e.g., replace with XXX) where the value is not needed.

### Generalization

Replace precise values with ranges.

## Verification

After anonymization, verify that re-identification is not possible from the remaining data.
