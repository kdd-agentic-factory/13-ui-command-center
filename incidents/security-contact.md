# Security Contact

## Reporting a Security Issue

To report a security issue in this ecosystem:

1. Do not create a public GitHub issue.
2. Contact the platform owner directly.
3. Include: description, severity assessment, affected components, evidence.

## Escalation

For SEV-1 incidents: escalate immediately to the platform owner and architecture owner.

## Response SLA

- SEV-1: immediate response
- SEV-2: within 4 hours
- SEV-3: within 2 business days
