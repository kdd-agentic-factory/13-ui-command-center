# Secure Design Checklist

- [ ] Trust boundaries are defined.
- [ ] Data flows are documented.
- [ ] Threat model is completed.
- [ ] Sensitive data is identified and classified.
- [ ] Least privilege is applied to all components.
- [ ] Authentication and authorization are designed.
- [ ] Secrets are never hardcoded.
- [ ] Audit events are planned for critical actions.
- [ ] Human approval gates are designed for critical actions.
- [ ] Input validation is designed at all entry points.
